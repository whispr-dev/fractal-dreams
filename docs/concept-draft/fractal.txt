🔁 1. Fractal Compression Basics
Fractal image compression (and theoretical fractal data compression) works like this:

You divide the input (image, signal, etc.) into range blocks (target regions).

You search over domain blocks (larger pool of regions) to find affine-transformed matches.

Instead of storing raw pixels/data, you store the transform parameters that, when iterated, reconstruct the input.

The magic is in exploiting self-similarity and recursive mapping.

❌ Problem:
What if your algorithm only finds the whole as self-similar to itself?

Then your only "match" is the full domain = full range.

You can't subdivide, and therefore can't recurse — defeating compression.

🧩 Solutions in Real Fractal Compression:
1. Minimum Entropy Threshold
Real fractal encoders don't look for perfect self-similarity.

Instead, they allow approximate matches within some error threshold ε.

Even slight gradient changes, noise, or variation allows a small patch to be encoded as a transform of a larger one.

So you force a subdivision unless error is negligible.

Analogy: Like JPEG allows DCT quantization errors, fractal compression allows transform mismatches up to a tiny bit.

2. Forced Subdivision / Quadtree Partitioning
If a region can’t be matched sufficiently well, it’s subdivided (e.g., into quadrants).

This ensures that eventually you hit blocks small enough to match something well enough.

This solves your issue: the "whole image" match won't be allowed unless it's losslessly exact and entropic entropy = 0.

3. Domain Pool Restriction
You don’t compare the entire image to itself.

You compare each block only to a limited set of candidate blocks.

These are usually downsampled, transformed, or filtered to lower dimensional space — increasing match probability.

4. Gradient and Feature Analysis
To avoid whole-image traps, the encoder often uses feature detection (edges, gradients, textures) to guide which parts of the image might match others.

If two regions share a gradient structure, even if pixel values differ slightly, they’re considered "similar enough."

import torch
from transformers import GPT2LMHeadModel, GPT2Tokenizer

model_name = "gpt2"
model = GPT2LMHeadModel.from_pretrained(model_name)
tokenizer = GPT2Tokenizer.from_pretrained(model_name)
model.eval()

def generate(prompt, max_length=100):
    inputs = tokenizer(prompt, return_tensors="pt")
    with torch.no_grad():
        outputs = model.generate(
            **inputs,
            max_new_tokens=max_length,
            do_sample=True,
            top_k=50,
            top_p=0.95,
            temperature=1.0,
        )
    return tokenizer.decode(outputs[0], skip_special_tokens=True)

def extract_transform(before, after):
    """
    Naive transform: get the added suffix from model's response.
    """
    if after.startswith(before):
        return after[len(before):].strip()
    return after

def fractal_dream(seed, iterations=10):
    current = seed
    history = [seed]
    print(f"[0] {seed}")

    for i in range(1, iterations + 1):
        full = generate(current)
        delta = extract_transform(current, full)
        current += " " + delta
        history.append(current)
        print(f"[{i}] {delta}")

    return history

if __name__ == "__main__":

    fractal_dream(seed, iterations=10)

4.1 Recursive Token Self-Mirroring
One of the simplest yet most revealing hallucination engines we propose is Recursive Token Self-Mirroring — a fractal-inspired mechanism that bootstraps symbolic complexity through self-application of model-generated deltas.

Inspired by techniques from fractal image compression and iterative function systems (IFS), this method treats the output of a language model not as a terminal prediction, but as a generative transform — one that can be recursively applied to itself. Rather than seeking a stable or bounded completion, we explicitly destabilize the loop to see where the model "wants to go" when it amplifies its own ideas.

Algorithm Description
We define the core algorithm as follows:

Let the model be a function M that maps an input token sequence Sₙ to an output string Rₙ = M(Sₙ).
Define a delta function Δ(Sₙ, Rₙ) which extracts the new symbolic content Tₙ = Rₙ \ Sₙ.
Then:
Sₙ₊₁ = Sₙ + Tₙ
Iterate for n = 0…N.

This process may be interpreted as:

Feed the model a seed sequence S₀

Generate a continuation R₀

Extract the appended content T₀ = Δ(S₀, R₀)

Form the new input S₁ = S₀ + T₀

Repeat steps 2–4 to obtain a growing recursive text

Unlike classical text generation which stops at a predicted endpoint, this process treats the delta between input and output as a reusable symbolic increment — a fractal generator in the space of tokens.

We implemented this method using GPT-2, a freely available pretrained transformer model, using nucleus sampling and a temperature-controlled decoder. The extract_transform() step uses naive prefix-differencing to obtain the suffix, though more sophisticated edit-distance deltas may be used for greater semantic control.

4.2 Activation Space Gradient Climbing
While recursive token self-mirroring treats the model’s output delta as the generative core, this next technique treats the model’s internal state — specifically its layer activations — as the target of generation.

Inspired by DeepDream’s approach to image synthesis, this method performs gradient ascent in the activation space of a transformer, seeking to amplify latent features that correspond to meaningful or novel concepts within the model.

🧠 Concept: Dreaming in the Attention Spine
Every transformer has a hierarchy of layers, each containing attention heads and feedforward projections. As input flows through these layers, rich intermediate representations — the hidden states — form, encoding semantic, syntactic, and conceptual structures.

In standard inference, the model outputs text optimized for next-token likelihood.
In our approach, we instead ask:

“What input tokens most activate a specific internal representation — and how can we tweak the input to maximize that?”

This results in hallucinogenic prompts — inputs crafted to provoke maximal activation of internal feature detectors.

M(x) be the transformer model

Hᵢ(x) be the activation at layer i for input x

L(Hᵢ(x)) be a scalar loss measuring how "excited" the activation is

x be a token embedding vector (continuous)

1. Start with random or seed input tokens x₀
2. Forward pass: get activation Hᵢ(x₀)

4. Backpropagate ∇L wrt x₀

from transformers import GPT2LMHeadModel, GPT2Tokenizer
import torch
from torch.nn.functional import mse_loss

model = GPT2LMHeadModel.from_pretrained("gpt2", output_hidden_states=True)
tokenizer = GPT2Tokenizer.from_pretrained("gpt2")
model.eval()

def dream_tokens(seed_text, target_layer=8, steps=20, lr=0.05):
    input_ids = tokenizer(seed_text, return_tensors='pt')['input_ids']
    input_embed = model.transformer.wte(input_ids).detach().clone().requires_grad_(True)
    
    optimizer = torch.optim.Adam([input_embed], lr=lr)

    for i in range(steps):
        optimizer.zero_grad()
        outputs = model(inputs_embeds=input_embed)
        hidden_states = outputs.hidden_states[target_layer]
        loss = -hidden_states.norm()  # maximize norm (DeepDream style)
        loss.backward()
        optimizer.step()
        print(f"Step {i} | Activation: {-loss.item():.4f}")

    # Project back to tokens
    final_embed = input_embed.detach()
    logits = torch.matmul(final_embed, model.transformer.wte.weight.T)
    tokens = torch.argmax(logits, dim=-1)
    return tokenizer.decode(tokens[0])

4.3 Latent Vector Transform Composition
This hallucination engine takes a further abstract step away from token space and enters the continuous latent landscape of the transformer. from transformers import GPT2Tokenizer, GPT2Model
import torch
import torch.nn.functional as F

model = GPT2Model.from_pretrained("gpt2")
tokenizer = GPT2Tokenizer.from_pretrained("gpt2")
model.eval()

def get_embedding(text):
    inputs = tokenizer(text, return_tensors='pt')
    with torch.no_grad():
        outputs = model(**inputs)
        hidden = outputs.last_hidden_state
        return hidden.mean(dim=1)  # mean-pooled embedding

def decode_embedding(embedding, vocab_embeds):
    # Cosine similarity match
    scores = F.cosine_similarity(embedding, vocab_embeds, dim=1)
    top_tokens = torch.topk(scores, k=10).indices
    return [tokenizer.decode([i]) for i in top_tokens]

def latent_chain(a_text, b_text, steps=5):
    e_a = get_embedding(a_text)
    e_b = get_embedding(b_text)
    delta = e_b - e_a

    vocab_embeds = model.wte.weight.detach()

    current = e_b
    for i in range(steps):
        current = current + delta
        decoded = decode_embedding(current, vocab_embeds)
        print(f"[{i+1}] {decoded[:3]}...")  # show top-3 guesses
